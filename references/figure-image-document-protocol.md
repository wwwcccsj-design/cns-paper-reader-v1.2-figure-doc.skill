# 带 Figure 图文档交付协议

## 目标

生成论文精读文档时，不只输出文字，还要把论文中的 Figure 图片、页面截图或用户上传的 figure 图插入到 Word `.docx` 文档中，让读者能够“边看图边读解读”。

## 强制规则

1. **Figure 图片优先**：能提取单张 Figure 时，插入单张 Figure；不能裁图时，插入包含该 Figure 的 PDF 页面截图。
2. **图片在前，解读在后**：每个 Figure 小节必须先出现图片/截图/占位，再出现整图任务和逐 panel 解读。
3. **图注必须保留**：Figure 原英文图题和图注应被摘录或概括，随后给中文解释。
4. **图片来源要标注**：例如“来源：PDF 第 5 页，Figure 2”。
5. **不可见要说明**：如果没有图片、图太糊、图注缺失或无法裁取，必须在文档中写明限制，不能假装看到了图。
6. **不允许空泛图解**：每个 panel 都要回答“做了什么、怎么做、为什么做、结果、意义、风险”。

## Word 文档中的推荐排版

```markdown
## Figure X：中文标题

[Figure X 图片，宽度适配页面]

> 原图题 / 图注核心信息：……

### 整图任务
这张图在全文中用于……

### 整图实验设计
作者先……然后……最后……，目的是……

### 逐 panel 解读
[按 Fig.Xa、Fig.Xb、Fig.Xc 展开]

### Figure X 小结
这张图把论文证据链推进到了……
```

## 图片提取策略

- 如果输入是 PDF：读取 figure 所在页，优先截图或裁取对应区域。
- 如果输入是用户上传的 Figure 图片：直接插入该图片。
- 如果输入只有文字/摘要：文档中创建 Figure 占位，并明确“未提供原图，以下基于图注/正文解读”。
- 如果 Figure 分散在补充材料：主文档先解释主图，补充图放入“Supplementary Figure 证据补充”章节。

## 图片失败占位格式

```markdown
[Figure X 图片占位]
无法插入原因：[PDF 无法解析 / 未提供原图 / 图片分辨率不足 / 当前环境无法导出图片]
可用信息来源：[图注 / 正文结果段落 / 用户截图]
```

## 质量底线

最终文档必须让读者做到：

- 看到 Figure 图片。
- 知道每个 panel 是什么实验。
- 知道作者为什么做这个实验。
- 知道结果从图上怎么看出来。
- 知道这个结果支持了论文哪一步结论。
- 知道这个证据有没有漏洞。
